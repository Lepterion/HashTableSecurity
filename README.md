# Secure Hash Table (SHT) & In-Memory Vault 🛡️

## Опис проєкту
Цей проєкт — це кастомна реалізація системи зберігання ключів та значень, спроєктована з акцентом на безпеку даних в оперативній пам'яті (In-Memory Protection). 

Стандартні структури даних (наприклад, звичайні хеш-таблиці) зберігають інформацію у відкритому вигляді (plaintext). Це робить чутливі дані вразливими до сканування пам'яті (Memory Dumping / RAM Scanning) за допомогою дебагерів або шкідливого ПЗ. Наше рішення розв'язує цю проблему, обфускуючи дані "на льоту".

## 🎯 Threat Model (Модель загроз)
Якщо зловмисник отримує доступ до адресного простору нашого процесу, він може легко знайти паролі чи токени. **Наша мета:** зробити так, щоб дамп пам'яті повертав лише зашифроване "сміття" замість реальних значень, при цьому зберігши швидкість доступу $O(1)$.

## ⚙️ Архітектура та Протокол захисту
Ми використовуємо **Динамічну In-Place Обфускацію** з такими ключовими компонентами:

1. **Динамічний сесійний ключ:** Генерується при кожному запуску програми. Навіть якщо зберегти дамп пам'яті двічі, дані виглядатимуть по-різному.
2. **In-Place XOR маскування:** При записі даних, кожен байт значення шифрується унікальною маскою: `Mask = Session_Key ^ Hash(Key)`. Шифрування та дешифрування відбувається прямо в оригінальному масиві без виділення додаткової пам'яті.
3. **Secure Wiping (Захист від витоку):** При видаленні вузла пам'ять не просто звільняється, а примусово затирається нулями. Використовується вказівник `volatile`, щоб компілятор не проігнорував цикл очищення під час оптимізації.

## 🗂️ Структура репозиторію
Команда розподілила задачі наступним чином:
* `the_code.cpp` — Ядро протоколу безпеки (`SecurityProtocol` та структура `SecureNode`).
* `test.cpp` — Автоматизовані тести (Security Verification), які напряму сканують сиру пам'ять, щоб довести відсутність оригінальних рядків, а також перевіряють коректність In-Place дешифрування та затирання пам'яті.
* `test_app.cpp` — Міні-застосунок "Secure Vault". Консольна демонстрація, яка наочно показує різницю між тим, що бачить хакер (Raw RAM Dump), і тим, що отримує легальний користувач через API.
